Step 1: Create your organization

The first step is to create an organization which will be a container for all of your services. If you are not logged in - you will be prompted to first login or register.

Create an organization then continue with Step 2.

Step 2: Create your first service

A service in apidoc represents a full REST service. Each service is fully described in a single file. By convention, we name these files 'api.json' and store them in the root of the project that defines the server that implements the service (e.g. the root directory of your source repository).

The easiest way to get started is download an example api.json file and then edit it - this api.json file is the description of the REST API for apidoc itself.

When you are ready - Click "@lib.Util.AddServiceText" from the upper right corner of your organization's screen to upload your api.json file.

Step 3: View your beautiful documentation!

For many users you are now done - apidoc will present beautiful documentation for your REST service. If you distribute the URL with the version number 'latest' - you will always have a link to the most current version of your service.

Step 4: Download a client library

When viewing your service, The left hand nav bar will have a section titled Clients. Take a look at the Ning Async Http Client 1.8 and the Ruby client, for example, to see how they look.

Each client is presented as a standalone single file which should make it easy to drop directly into your project or to publish an artifact to a server for sharing.

Step 5: Explore

From your organization, explore the settings to define your package name for generated clients and to consider marking your organization public.
